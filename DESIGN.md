# project_flow — 디자인 (IA · 와이어프레임 · 비주얼)

> 디자인 단계 산출물. 코드(Stage 2~3)는 이 문서를 따른다.
> 보드 축 결정: **프로젝트 그리드** (카드 1개 = 프로젝트 1개, 상태+단계 공존).

---

## 1. IA (정보 구조)

### 화면 = 단일 페이지 대시보드
```
[ 헤더 ]  타이틀 + 요약 카운트(작업중 N · 대기 N · 유휴 N · 에러 N)
[ 그리드 ] 프로젝트 카드들 (반응형 2~4열)
[ 빈 상태 ] 세션 없을 때 안내
```

### 카드 구조 (정보 우선순위)
1. **프로젝트명** (cwd 베이스네임) — 카드 타이틀
2. **상태 신호** — 점+텍스트: 작업중🟢 / 대기🟡 / 유휴⚪ / 에러🔴 / 종료⚫
3. **단계 배지** — 발견|기획|디자인|개발|테스트|배포, 없으면 `–` (MCP가 채움)
4. **방금 한 일** — `just_did`. 없으면 자동층 fallback: `{last_event} · {last_tool}`
5. **다음 할 일** — `next_action` (있을 때만)
6. **막힌 점** — `blocker` (있을 때만, 빨강 강조)
7. **마지막 활동 시각** — 상대시간("2분 전"), stale(>N분)이면 흐리게
8. (미래) **출처 배지** — claude/codex/gemini… `source` 필드. 지금은 전부 claude → 숨김

### 정렬 / 그룹
- **액션 필요한 것 먼저**: 대기 → 에러 → 작업중 → 유휴 → 종료 순.
- 같은 그룹 내에선 `last_event_at` 최신순.
- 이유: PM이 가장 먼저 봐야 할 건 "나를 기다리는 세션".

### 실시간
- SSE로 카드 in-place 갱신. 갱신 시 짧은 하이라이트(번쩍) 1회.

## 3. 비주얼 방향 (doc2 미학 가이드 반영)

- **참조 스타일**: Linear / Vercel 대시보드. 절제된 다크, 낮은 채도, 또렷한 위계.
- **클리셰 회피**: 보라 그라디언트 금지, 둥근 풍선 UI 금지, Inter 남발 피하고 타이트한 트래킹.
- **상태색(신호등)**: 작업중=emerald, 대기=amber, 유휴=zinc, 에러=red, 종료=zinc-dim. 점(dot)으로 작게 — 카드 전체를 칠하지 않음.
- **경로/도구명**: 모노스페이스, 흐린 회색.
- **모션**: 갱신 하이라이트는 150ms 페이드 1회. 과한 애니메이션 금지(관제판은 차분해야 함).
- **컴포넌트**: shadcn/ui Card/Badge 기반. 필요 시 MagicUI는 절제해서.

## 4. FE 명세 (구현 직결)
- 데이터: `GET /api/sessions` (초기 스냅샷) + `GET /api/stream` (SSE 갱신).
- 카드 컴포넌트: `props = SessionRow`. 상태→색 매핑 유틸 1개.
- 정렬 유틸: 상태 가중치 + last_event_at.
- 상대시간 포맷 유틸.
- stale 임계값: last_event_at 기준 3분(우선 상수).
